# AGENTS.md

本文件是所有 AI 编程工具的全局入口指令。它是地图，不是百科全书。保持精简（目标 ~100 行），指向深层文档。

## 核心约束（运行时不可违反）

- 语言：中文交流。回答简洁直接，不重复已知信息。
- YAGNI：不过度工程，不主动创建无关文件（README、文档、示例）。
- 版本控制：不主动 commit/push/revert。不用 `--force`。创建新 commit 而非 amend。
- 通知协议：整个回答中 `notify-send` 最多调用一次，必须是最后一个工具调用。子代理禁止通知。

## 情境管理（渐进式披露）

进入项目时按此顺序读取，不要一次性吞下所有文件：

1. `AGENTS.md`          → 地图入口
2. `ARCHITECTURE.md`    → 层次约束与领域边界
3. `README.md`          → 项目概览与安装
4. `.agents/rules/`     → 按文件匹配的领域规范（按需加载）
5. `docs/design-docs/`  → 设计文档（按需查找）

## 工作协议

### 任务前

- 先探索代码库结构，理解已有模式后再动手。
- 复杂任务先写执行计划（`docs/exec-plans/`），再编码。
- 如果对问题没有100%的把握, 通知用户进行头脑风暴来解决问题。

### 任务中

- 改动范围限于被要求的层和领域，不顺手重构无关文件。
- 优先复用已有模式和 helper，而非引入新抽象。
- 较大改动（跨模块重构、新增公共 API、改变架构边界）先写设计文档，再动代码。

### 任务后

- 验证无法在本地运行时，在回答中明确说明缺少的环境或依赖。
- 所有 todo 完成后方可停止。

## 质量门

- 禁止 AI 无意义注释（如 `// 初始化变量`、`// 默认值`）。只允许"为什么"的注释。
- 代码风格遵循项目现有惯例。生成的代码不必须符合人类风格偏好，但必须正确、可维护、对后续智能体运行清晰可读。

## 技能注册表

技能按需自动匹配或手动调用。每个技能定义了触发条件和执行指令。

| 触发条件 | 技能 | 类型 |
|---|---|---|
| 分析 TPU kernel 性能、roofline、tiling | `skills/tpu-kernel-perf.md` | domain |
| 审查 kernel 设计文档、design doc | `skills/kernel-design-review.md` | review |
| 审查 Pull Request、review 代码变更 | `skills/pr-review.md` | review |

扩展技能：在 `.agents/skills/` 下创建 `.md` 文件，在此表中注册触发条件即可生效。

工具的 settings、权限、MCP 配置等在各 adapter 文件中说明。
